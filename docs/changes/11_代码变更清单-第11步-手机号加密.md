# 代码变更清单 · 第 11 步 · 手机号加密（AES-GCM + SHA256 pepper 迁移）

---

## 加密工具层

- `pkg/crypto/phone.go`（新增）
  · 三个函数：`EncryptPhone`（AES-256-GCM，StdBase64 输出）/ `DecryptPhone`（反向）/ `HashPhone`（SHA256+pepper）
  · 设计要点：key 为空字符串时 Encrypt/Decrypt 均为 no-op，保证 dev 向后兼容；nonce 每次 `io.ReadFull(rand.Reader)` 随机生成，满足 IND-CPA 属性（同一明文两次加密结果不同）；GCM auth-tag 失败（明文电话号码 ≤15 字符，base64 后 <12 字节 nonce）天然成为迁移脚本的幂等探针

- `pkg/crypto/phone_test.go`（新增）
  · 10 个单元测试：round-trip 正确性；两次加密产生不同密文；空 key 返回明文；明文手机号用真实 key 解密必然失败；错误 key 解密失败；pepper 效果；无 pepper 等价 SHA256；确定性；输出 64 位小写 hex
  · 设计要点：`TestDecryptRejectsPlaintextWhenKeySet` 验证了迁移脚本依赖的核心假设——AES-GCM 不会把数字字符串误判为合法密文

- `pkg/config/config.go`（修改）
  · 新增 `EncryptionConfig` 结构体（`PhoneKey string` / `PhonePepper string`）
  · `Config` 根结构体追加 `Encryption EncryptionConfig` 字段
  · `validate()` 追加校验：`PhoneKey` 非空时必须是 64 位合法 hex 字符串（32 字节 AES-256 key）；允许为空（dev 降级）
  · 设计要点：不在 `registerDefaults()` 里设默认值（空串就是正确默认），`validate()` 校验格式而非存在性，prod 通过 env `APP_ENCRYPTION_PHONE_KEY` 注入

- `configs/config.yaml`（修改）
  · 追加 `encryption:` 段（`phone_key: ""` / `phone_pepper: ""`）
  · 注释说明 dev 空值行为、prod 注入方式、生成命令（`openssl rand -hex 32`）

- `configs/config.prod.yaml`（修改）
  · 追加 `encryption:` 段，两字段用 `<PLACEHOLDER>` 格式提示运维替换
  · 设计要点：占位符风格与 JWT secret、OSS key 等保持一致，方便 docker-compose 模板扫描替换

## 业务层

- `pkg/errcode/errcode.go`（修改）
  · 追加 480101 `ErrEncryptPhone`（500）/ 480102 `ErrDecryptPhone`（500）
  · 设计要点：两者均为内部错误，不返回给 HTTP 调用方；480102 在 `GetMyProfile` 降级路径里只打 warn 日志，供 Prometheus 通过 errcode 告警

- `internal/service/user_service.go`（修改）
  · **import**：追加 `"cooking-platform/pkg/crypto"`；移除因 `hashPhone` 迁移而变得冗余的包引用（实际 `sha256`/`hex` 仍被 `hashIP` 使用，保留）
  · **struct**：`UserService` 追加 `encCfg config.EncryptionConfig` 字段
  · **NewUserService**：签名追加 `encCfg config.EncryptionConfig` 参数（插在 `ossCfg` 和 `rlCfg` 之间）
  · **hashPhone**：从包级函数改为 `UserService` 方法，委托 `crypto.HashPhone(phone, s.encCfg.PhonePepper)`；`SendCode` 和 `Login` 两处调用点同步改为 `s.hashPhone(phone)`
  · **Login**：注册新用户前调用 `crypto.EncryptPhone(phone, s.encCfg.PhoneKey)`；加密失败返回 `errcode.ErrEncryptPhone`；`defaultNickname(phone)` 仍用明文 phone（在加密前执行）
  · **GetMyProfile**：`maskPhone(user.PhoneEncrypted)` 改为先 `crypto.DecryptPhone` 再 `maskPhone(plain)`；解密失败打 warn + `plain=""` 降级（maskPhone("") 返回空串，不 panic）
  · 设计要点：`hashIP` 保持包级函数（IP 无 pepper 需求），不随 `hashPhone` 一起迁移，两函数职责分离清晰

- `cmd/server/main.go`（修改）
  · `NewUserService` 调用处补 `cfg.Encryption` 参数（第 7 个位置，在 `cfg.OSS` 和 `cfg.Ratelimit` 之间）

## 数据迁移工具

- `cmd/migrate-phone/main.go`（新增）
  · 独立 binary，复用 `config.Load()` 读 `configs/config.yaml` + 环境变量；支持 `--dry-run` flag
  · `migrate()` 函数：游标分页（`WHERE id > lastID ORDER BY id LIMIT 200`），比 OFFSET 稳定；逐行 `computeUpdates()` 判断；每行独立 `UPDATE`，无全局事务，崩溃可重跑
  · `computeUpdates()` 核心逻辑：`DecryptPhone` 失败 → 明文 → 需加密；成功 → 已密文 → 只判断 phone_hash 是否需 pepper 更新；两者均无变化时跳过（幂等）
  · dev 模式（key="" pepper=""）：进入 main 直接打印 "nothing to migrate" 并 exit 0，CI 不报错
  · 设计要点：`initDB` 用独立的低连接数配置（MaxOpen=5），迁移是批量只写，不需要应用级连接池规格；`--dry-run` 下只打印行级变更意图，不执行 UPDATE
