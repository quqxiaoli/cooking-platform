## 缓存层

- `internal/cache/event_dedup_cache.go`（新增）
  · A6 IDEMP-01 的核心实现。SETNX + cfg.Cache.DedupTTL，key 模板 `dedup:<topic>:<event_id>`，topic 入 key 防跨主题 EventID 碰撞。
  · `SeenOrMark(ctx, topic, eventID) (alreadySeen bool, err error)` 是唯一对外方法——原子 check-and-mark 一次完成，避免"先 Get 后 Set"竞态。
  · 设计要点：fail-open 策略（Redis 故障返 false + err，调用方处理），rare double-count 远优于 silent data loss；emptyEventID 直通不阻塞。

- `internal/cache/user_cache.go`（修改）
  · A5 USER-03。构造函数加 `smsDailyTTL time.Duration` 参数（cfg.Cache.UserSMSDailyTTL）。
  · `IncrementAndCheckSMSPhoneDaily` / `IncrementAndCheckSMSIPDaily` 的 24h 硬编码改成 `c.smsDailyTTL`。

## 接入层

- `internal/handler/health.go`（修改）
  · A2 INFRA-08 + SYS-08。Readiness 失败路径从手拼 `c.JSON(503, gin.H{...})` 改为 `response.Unavailable(c, checks)`，envelope 与全 API 统一；data 字段仍是 checks map。

- `internal/middleware/cors.go`（修改）
  · A5 CORS-01。整文件重写。`CORS(cfg config.CORSConfig) gin.HandlerFunc`：wildcard 短路；显式列表用 origin map O(1) 查 + 写 `Access-Control-Allow-Origin: <echoed>` + `Vary: Origin`；methods/headers/expose-headers 从 cfg join 渲染。

## 业务层

- `internal/service/follow_service.go`（修改）
  · A5 FOLLOW-01。删 `maxFollowing / defaultFollowListSize / maxFollowListSize` 三 const；结构体加 `maxFollowing` / `defaultListSize` / `maxListSize` 三字段；构造函数加 `cfg config.FollowConfig`；`clampFollowListSize` 从函数提为方法以便读 struct 字段。
