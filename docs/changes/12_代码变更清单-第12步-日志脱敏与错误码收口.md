# 代码变更清单 · 第 12 步 · 日志脱敏 + 错误码体系最终收口

---

## 工具层

- `pkg/crypto/mask.go`（新增）
  · 接口：`MaskPhone(phone string) string` / `MaskToken(token string) string`
  · 设计要点：从 user_service 私有函数提升为 pkg 层公开 API，使任意模块（audit、follow 等）都能引用，避免各自实现格式不一致。MaskToken 对 ≤8 位 token 全遮——短 token 的前 8 位没有关联价值，全遮更安全。

- `pkg/crypto/mask_test.go`（新增）
  · 7 个 case，覆盖边界长度、MaskToken 的 ≤8 分支和 >8 分支

- `pkg/logger/fields.go`（新增）
  · 接口：`MaskedPhone(phone string) zap.Field` / `MaskedToken(token string) zap.Field`
  · 设计要点：把"脱敏"嵌入 zap.Field 类型名，调用方读代码时无需追进实现就能确认安全性。属于"安全原语"模式。

---

## Middleware 层

- `internal/middleware/security.go`（新增）
  · 四个 OWASP 推荐响应头：X-Content-Type-Options(nosniff) / X-Frame-Options(DENY) / X-XSS-Protection(1;mode=block) / Referrer-Policy(strict-origin-when-cross-origin)
  · 设计要点：无 dev/prod 分支，两个环境行为一致，避免测试路径和生产路径差异掩盖安全问题。

- `internal/middleware/logger.go`（修改）
  · 改动：`query` 字段由 `c.Request.URL.RawQuery` 改为 `sanitizeQuery(c.Request.URL.RawQuery)`
  · `sanitizeQuery`：url.ParseQuery 解析后遍历 map，对 phone/mobile/code/token/access_key/secret/password 参数 value 替换为 `***`；解析失败兜底返回 `[unparseable]`。
  · 设计要点：防御层，不依赖调用方记住脱敏。当前所有 phone 路由都走 JSON body，query string 里不会出现 phone，但提前封锁防止未来路由误设计导致泄露。

- `internal/middleware/logger_test.go`（新增）
  · TestSanitizeQuery：6 个 case，覆盖 phone/code/token 脱敏、非敏感参数保留、空串、大小写混用的 key
