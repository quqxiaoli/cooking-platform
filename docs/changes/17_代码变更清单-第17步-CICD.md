# 代码变更清单 · 第 17 步 · GitHub Actions CI/CD

## CI/CD 基础设施层

- `.github/workflows/pr.yml`（新增）
  · 触发条件：PR 目标分支 main。三步检查：go vet（静态分析基础）→ staticcheck（更严格的 vet 超集，安装 @latest）→ go test -v -race -count=1（含 services）。
  · services 块声明 mysql:8.0（healthcheck ping）+ redis:7.2（healthcheck ping），CI 环境注入 TEST_MYSQL_DSN / TEST_REDIS_ADDR 两个 env var 触发集成测试。
  · 设计要点：`go-version-file: go.mod` 自动读取版本，setup-go cache=true 缓存模块，不在 workflow 里硬写 Go 版本号，版本升级只改 go.mod 一处。

- `.github/workflows/build.yml`（新增）
  · 触发条件：push 到 main 分支（不含 PR 本身）。两步：go build ./...（验证整个 module 可编译）+ docker build -t cooking-platform:ci .（验证 Dockerfile 没坏）。
  · 设计要点：不 push 镜像，只做 smoke build。push 由 release.yml 负责，两个职责分离。

- `.github/workflows/release.yml`（新增）
  · 触发条件：push tags v*。三步：GHCR 登录（secrets.GITHUB_TOKEN，无需额外配置）→ docker/metadata-action 从 tag 提取版本号（type=semver 产出 `1.0.0`，type=raw 额外打 `latest`）→ docker/build-push-action 构建并推送。
  · 设计要点：permissions.packages=write 是 GHCR push 的必要权限，须在 job 级声明；metadata-action 把 "git tag → Docker tag" 的映射逻辑从 shell 脚本里抽出来，标准化且支持预发布版本（v1.0.0-rc.1 → 1.0.0-rc.1）。

## 测试层

- `internal/integration/ping_test.go`（新增）
  · 两个集成测试：TestMySQLPing（sql.Open + db.Ping()）+ TestRedisPing（redis.NewClient + PING）。
  · 设计要点：env-var-guard 模式——env var 不存在时 t.Skip()，不 t.Fatal()，保证本地 go test ./... 能正常通过（SKIP 不算失败）。CI 里 services 起来并注入 env var 后才真正执行。
  · 选择 internal/integration/ 包：明确标识为集成测试，与单元测试分目录，go test ./internal/integration/... 可独立运行，也在 ./... 通配符范围内。

## 验证脚本

- `scripts/verify_step17.sh`（新增）
  · 8 段验证：workflow 文件存在性 → YAML 语法（ruby stdlib yaml）→ 触发器配置 → services 块完整 → go build/vet → 集成测试文件 → 单元测试 → 集成测试本地 SKIP。
  · 设计要点：YAML 验证用 `ruby -e "require 'yaml'"` 而非 python3 yaml，因 pyyaml 在 macOS 上不是默认安装包；ruby stdlib 的 Psych::SafeLoader 等价于 PyYAML.safe_load。
