## 数据层

- `migrations/000006_create_audit_logs_table.up.sql`（新增，后修改）
  · 只追加不更新的合规审计日志表；无外键约束（post/user 软删除后记录独立存在）；raw_response TEXT 存阿里云原始 JSON；idx_post_id + idx_created_at 支持帖子历史查询与时间段导出
  · **修改**：移除 `raw_response TEXT NOT NULL DEFAULT ''` 中的 `DEFAULT ''`——MySQL 8.0 严格模式（sql_mode=STRICT_TRANS_TABLES）禁止 TEXT/BLOB 列设置 DEFAULT 值（Error 1101），5.x 宽松模式静默忽略此错误，升至 8.0 才暴露；修复后 dirty state 需 `migrate force 5 && migrate up` 重建

- `migrations/000006_create_audit_logs_table.down.sql`（新增）
  · `DROP TABLE IF EXISTS audit_logs`

- `internal/model/audit_log.go`（新增，后修改）
  · GORM model，无 `UpdatedAt` 字段（行不可变语义，防止误调 Save 覆盖历史）；`TableName()` 返回 `"audit_logs"`
  · **修改**：RawResponse GORM tag 同步去掉 `default:''`，与 migration 变更保持一致

- `internal/repository/audit_repository.go`（新增）
  · interface `AuditRepository` 只暴露 `Create`，刻意不暴露 Update/Delete，让 interface 本身成为"只追加"约束的一部分

- `internal/repository/post_repository.go`（修改）
  · 接口末尾追加 `UpdateAuditStatus(ctx, postID, auditStatus, isVisible, remark)`；实现用 `map[string]interface{}` 而非 struct，确保 `is_visible=0`（拒绝时）不被 GORM zero-value 机制静默跳过；WHERE 子句精确到主键，O(1) 操作

## 外部依赖层

- `pkg/config/config.go`（修改）
  · `SMSConfig` 追加 `AccessKeyID / AccessKeySecret`（env 注入，prod 必填）；新增 `AuditConfig`（provider / AccessKey / Region / MockResult）；`validate()` 补充 aliyun 两个 provider 的必填字段校验；`registerDefaults()` 追加 audit 三个默认值

- `pkg/audit/doc.go`（修改）
  · 更新为反映三件套结构的包级文档

- `pkg/audit/auditor.go`（新增）
  · `Auditor` interface：`Review(ctx, ReviewRequest) (ReviewResult, error)`；`ReviewRequest` 含 PostID / AuthorID / Title / Content / ImageURLs（步骤图片已展平）；`ReviewResult` 含 Status(uint8) / Remark / Raw；`NewAuditor(cfg)` factory 按 provider 分发；`worstStatus` helper 取两个机审结果中更严格者（block > review > pass），供 AliyunAuditor 合并文本+图片扫描结论

- `pkg/audit/mock.go`（新增）
  · `MockAuditor` 读 `cfg.MockResult` 返回固定 verdict（"pass"→1 / "suspect"→2 / "reject"→4，其他默认 pass）；Raw 字段写 JSON blob 含 provider/post_id/status/timestamp，让 verify 脚本能断言 raw_response 非空

- `pkg/audit/aliyun.go`（新增）
  · `AliyunAuditor` 包装 `green.Client`；`Review` 先跑 TextScan（标题+正文合并），有图片则跑 `ImageSyncScan`（注：SDK 方法名为 `ImageSyncScan` 非 `ImageScan`，编译期发现）；两路结果取 `worstStatus`；suggestion 映射："block"→4 / "review"→2 / "pass"→1；Green API 返回非 200 code 视为错误；SDK client 创建一次复用（thread-safe）

- `pkg/sms/aliyun.go`（新增）
  · `AliyunSender` 包装 `dysmsapi.Client`；`SendCode` 构造 TemplateParam `{"code":"XXXXXX"}`；resp.Code != "OK" 即返回详细 error；Region 固定 cn-hangzhou（dysmsapi 全球统一入口）

- `pkg/sms/sender.go`（修改）
  · aliyun case 从 `return nil, fmt.Errorf("not implemented")` 改为 `return NewAliyunSender(cfg)`

## 业务层

- `pkg/errcode/errcode.go`（修改）
  · 追加 470xxx 段：`ErrAuditAPIFailed(470101)` + `ErrAuditWriteFailed(470102)`；两者仅用于结构化日志 errcode 字段，不返回给 HTTP 调用方（审核全异步，用户感知不到单次 API 失败）

- `internal/service/post_service.go`（修改）
  · `Create` 中 `IsVisible` 由 `PostVisible(1)` 改为 `PostInvisible(0)`，实现 PRD §9.2 Step 10 起的审核前不可见语义
  · `Create` 在 `publishPostEvent` 之后追加 `publishAuditEvent` 调用
  · 新增私有方法 `publishAuditEvent`：构造 `AuditEvent{AuditStatus=0}`（pending/提交信号）发往 `TopicAudit`；Remark/RawResponse 留空由 Consumer 填充；发布失败仅 Warn 不阻断请求

- `internal/consumer/audit_consumer.go`（新增）
  · 订阅 `TopicAudit`；`process` 方法：① 检查 AuditStatus==0（跳过非 0 事件，为未来人工审核预留）② FindByID + LoadSteps ③ `auditor.Review()` ④ `auditRepo.Create()`（先写合规日志） ⑤ `postRepo.UpdateAuditStatus()` ⑥ `feedCache.BumpFeedVersion()`（仅 MachinePass 时 bump）
  · fail-closed：API 失败时帖子保持 pending，不翻 visible；日志打 errcode=470101 供 Grafana 告警
  · `collectImageURLs` helper：cover_url（非空时）+ 各步骤 image_urls 展平为一个 slice
