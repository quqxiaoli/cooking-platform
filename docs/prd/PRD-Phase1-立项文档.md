## 1. 项目背景与立项动机

### 1.1 市场背景

中国在线美食内容市场正处于结构性增长期：

- **用户规模**：2025 年中国美食类短视频/图文用户超 **6 亿**，同比增长约 12%
- **年轻人下厨意愿提升**：Z 世代独居比例提高，「会做饭」成为重要的生活能力与社交资本
- **松弛感内容趋势**：2024 ~ 2025 年，「松弛感」「反焦虑」成为中国年轻人最高频的内容消费关键词，美食内容正在从「精致展示」向「真实记录」迁移
- **场景化需求爆发**：出租屋烹饪、露营料理、打包便当等细分场景在小红书、B 站均呈现高速增长，但缺乏专属的沉淀型内容平台

### 1.2 灵感来源

产品灵感源自日本动漫《摇曳露营》（ゆるキャン△）中角色使用的同类烹饪分享网站。该类网站的核心气质：

- **不焦虑**：没有流量压力，没有算法博弈，做了就发
- **有根基**：真实的烹饪过程，步骤是核心，不是摆拍
- **有场景感**：山野露营的炉火、冬日的车厢料理，场景赋予内容情绪温度

这正是国内年轻内容消费者日益渴望、却在现有平台上始终找不到归属感的产品形态。

### 1.3 核心问题陈述

> 当一个 25 岁的年轻人第一次在出租屋里认真煮了一锅汤， 他/她想分享的不是「食谱」，而是「那个下午」。 我们要做的，是能承载这种分享欲的地方。

## 2. 产品定位与核心价值

### 2.1 定位修订说明（v1.0 → v1.1 关键变化）

|维度|v1.0（已废弃）|v1.1（当前）|
|---|---|---|
|产品隐喻|「永不过期的食谱书」|「记录真实烹饪场景的社区日志」|
|内容框架|工具 × 信息（如何做）|场景 × 情绪（在哪里做、为什么做）|
|分类维度|菜系 / 食材|烹饪场景（出租屋 / 露营 / 减脂餐 / 家庭 / 快手菜等，共 8 个）|
|主力用户|22 ~ 40 岁家庭主妇为主|22 ~ 28 岁独立生活的年轻人为主|
|情感基调|专业、精准、可信赖|松弛、真实、有温度|
|对标参照|下厨房、Cookpad|《摇曳露营》里的那个网站 × VSCO日记感|

> **设计原则**：松弛感不等于不认真。我们的内容仍然要有真实的步骤、真实的过程图，烹饪根基不动摇。松弛感是「发布的门槛低、氛围不焦虑」，而不是「内容可以不认真」。

---

### 2.2 产品一句话定位

> **「不管是出租屋的深夜煮面，还是山野间的露营飘香——记录你每一次认真下厨的地方」**

### 2.3 核心价值主张（v1.1）

**对内容创作者（发帖者）：**

- 按场景发布，找到懂你的人，而不是在通用 Feed 里被淹没
- 步骤化发布工具专为烹饪设计，但不强迫「标准化」，露营锅煮的泡面也是值得记录的一餐
- 内容沉淀，不依赖算法，三年后你的「出租屋红烧肉」还能被人搜到
- 没有评论区的压力，只有点赞带来的正反馈

**对内容消费者（看帖者）：**

- 看到「和我一样的人，在和我一样的处境下」做出的饭——这种认同感是现有平台给不了的
- 场景标签帮助快速找到「出租屋单口灶能做的菜」而不是「精装厨房大菜」
- 步骤图文随时翻查，不需要暂停视频，适合真实操作场景

**对平台整体：**

- 用「场景」而非「算法」连接人，形成有归属感的垂直社区
- 内容天然长尾，可以持续被检索，不依赖实时流量维持生命力

---

### 2.4 产品情感关键词

```
松弛  ·  真实  ·  有温度  ·  可复现  ·  有场景感  ·  不焦虑
```

产品设计的一切决策都应接受这六个词的检验。

## 3. 目标用户画像

### 3.1 主力用户群 A：「独居/同居的第一代厨师」（25 岁核心圈层）

这是产品的核心增长飞轮，也是最高传播意愿的群体。

**基础属性**

|项目|描述|
|---|---|
|年龄|22 ~ 28 岁|
|生活状态|独居、与朋友合租、或刚组建的二人家庭|
|居住环境|城市出租屋，厨房通常设施有限（单口灶、小冰箱、锅具不全）|
|职业|应届生 / 工作 1 ~ 3 年 / 自由职业者 / 研究生|
|收入|月收入 5,000 ~ 15,000 元，对消费有选择性|
|设备|手机为主（iOS/Android 各半），偶尔用平板对照食谱|

**行为特征**

- 从「点外卖」到「偶尔自己做」再到「想认真做饭」的过渡期，内心有做饭的欲望但缺乏信心
- 在小红书/抖音上刷到好看的饭会心动，但那些菜「太复杂」或「厨房设备不够」，找不到「像我这样条件下能做的」
- 有强烈的「记录欲」，愿意把第一次成功烹饪分享出来，但不知道发在哪里合适
- 对「达人」内容有审美疲劳，更信任「看起来和我差不多的普通人」发的内容
- 对「出租屋」「下班后做」「一个人吃」这些标签有强烈的身份认同

**使用场景**

> 小雨刚毕业工作半年，住在上海一个 20㎡ 的出租屋里，厨房只有一个双口电磁炉。某个周日下午她决定认真做一顿，照着我们平台上一个「出租屋类」帖子里的步骤，用电磁炉做了西红柿鸡蛋面。成功了。她拍了 5 张过程图，在平台上发了一篇「出租屋 · 双口灶也能做好面」，3 天后收到 200 个点赞——第一次有人认可她的厨艺。她开始定期更新。

**痛点清单**

1. 现有食谱「默认有全套厨房设备」，不适配出租屋实际场景
2. 想分享但小红书/朋友圈太正式，感觉自己做的「不够好看」不值得发
3. 找不到「和自己条件一样的人」验证过的食谱
4. 下班累了还要做饭，需要「30 分钟内能完成」的方案

---

### 3.2 用户群 B：「认真的家庭厨师」（扩展圈层）

**基础属性**

|项目|描述|
|---|---|
|年龄|28 ~ 40 岁|
|生活状态|已婚或有长期伴侣，有固定居所和较完善的厨房|
|职业|各行业上班族、全职主妇/主夫|
|下厨频率|每周 4 ~ 7 次，有明确的菜品偏好和烹饪积累|

**与主力用户群的区别**

- 不太需要「被激励做饭」，已经有稳定的下厨习惯
- 对内容「可复现性」要求更高，对「松弛感」的共鸣相对弱
- 更关注食材选择、火候细节、家庭分量等实操细节
- 是平台「家庭类」内容的主要创作者，但不是最高传播节点

**使用场景**

> 阿芳每周末给家里做一顿大餐，她把自己改良过的「家常红烧肉」配方发在「家庭类」，步骤图拍得很认真，食材用量精确到克。她的内容被 25 岁的用户搜到，用在生日宴上。

---

### 3.3 用户群 C：「露营/户外烹饪爱好者」（精准垂直圈层）

**核心特征**

- 平台 IP 核心圈层，与《摇曳露营》品牌形象天然契合
- 内容质感好（户外光线、器具有美感），是平台对外传播的「门面内容」
- 社群属性极强，一个帖子容易在露营圈内部高速传播
- 有清晰的场景标签诉求（户外灶 / 卡式炉 / 荷兰锅 / 一锅料理）

**关键数据**（2025 估算）

- 中国露营用户规模约 4,000 万，年增长 20%+
- 露营 + 烹饪是露营活动中满意度最高的环节之一

---

### 3.4 创作者分层模型

```
T1 创作者（内容骨干）
  → 有稳定更新频率，内容质量高，是搜索 SEO 和社区调性的核心锚点
  → 约占发帖用户的 5%，贡献约 60% 的高质量内容

T2 创作者（活跃记录者）
  → 不定期发帖，主要记录自己的烹饪经历
  → 约占发帖用户的 30%，是社区活力的来源

T3 创作者（首次/偶发）
  → 发了一两篇，可能沉寂，也可能被激活成 T2
  → 约占 65%，是用户增长的主要来源
```

平台产品策略：**激活 T3 → 稳定 T2 → 激励 T1**，不盲目追求头部，保持社区的「普通人气质」。

## 4. 内容场景分类体系

### 4.1 设计原则

**场景优先，食材/菜系其次。**

与下厨房、美食天下的「川菜 / 粤菜 / 素食」分类体系不同，我们的一级分类是「你在哪里做」「用什么条件做」「为什么做」，而不是「做什么」。这是因为：

- 25 岁的核心用户找内容的第一个筛选条件是「我的厨房能不能做」，而不是「这道菜叫什么名字」
- 场景标签承载情绪认同（「出租屋」三个字对特定用户群是强共鸣信号，「减脂餐」对健身群体同理）
- 场景分类天然避开了和「菜谱 App」的正面竞争

> **设计备注**：8 个标签中，7 个属于「场所 / 时机 / 人数型」场景，「减脂餐」属于「目的型」场景。两类逻辑层次略有差异，但通过双标签机制可自由叠加（如「出租屋 × 减脂餐」），作为一级入口独立存在，架构自洽。

### 4.2 场景分类表（MVP 最终版，共 8 个）

|场景标签|英文 Key|核心定义|典型内容举例|主要用户群|
|---|---|---|---|---|
|出租屋|`rental`|设施有限（单 / 双口灶、无烤箱），1 ~ 2 人份|单口灶西红柿鸡蛋面、电饭煲料理|用户群 A（核心）|
|一个人的饭|`solo`|1 人份，强调「为自己好好做一顿」的仪式感|孤独美食家式独居料理|用户群 A|
|露营野炊|`camping`|户外场景，卡式炉 / 篝火 / 荷兰锅，自然背景|山顶泡面加料升级、荷兰锅炖肉|用户群 C|
|家庭厨房|`family`|有完整灶台、多口锅具的居家烹饪，家人份量|家常红烧肉、周末大餐、给父母做的菜|用户群 B|
|快手日常|`quick`|15 ~ 30 分钟内完成，效率优先|下班后 20 分钟番茄意面、微波炉快手菜|用户群 A|
|打包便当|`bento`|以「带走」为目的，保温 / 外观 / 便携为核心|健身便当、上班族盒饭、野餐便当|用户群 A / B|
|减脂餐|`diet`|低卡 / 高蛋白 / 轻食导向，健康目的明确|鸡胸肉沙拉、低卡代餐碗、健身增肌餐|健身 / 轻食用户|
|节气 / 节日|`seasonal`|依托节气或节日的应景料理|冬至饺子、清明青团、中秋月饼|用户群 B|

### 4.3 双标签机制

每篇内容支持同时打「场景标签」+「食材/口味标签」，实现两维检索：

```
场景维度（一级，必选一个）：出租屋 / 一个人的饭 / 露营野炊 / 家庭厨房 / 快手日常 / 打包便当 / 减脂餐 / 节气节日
食材/口味维度（二级，可多选）：素食 / 猪肉 / 海鲜 / 甜 / 辣 / 低卡 / 高蛋白 / ...
```

用户搜索时可按任意维度筛选，也可组合筛选（「出租屋 × 低卡」「露营 × 素食」「减脂餐 × 高蛋白」）。

### 4.4 内容时长维度（辅助标签）

|标签|含义|
|---|---|
|15 分钟内|超快手|
|30 分钟内|快手|
|1 小时内|中等|
|1 小时以上|慢炖/精工|

## 5. 竞品分析

### 5.1 重新定义我们的差异化

在修订后的定位下，我们的核心差异化从「结构化工具」升级为：

> **「场景认同感 × 松弛氛围 × 步骤可复现」三者的唯一结合体**

没有任何现有平台同时做到这三点：

- 小红书有场景感，但内容太精致，没有可复现性，也没有烹饪步骤的专属结构
- 下厨房有可复现性，但场景感为零，氛围沉闷，年轻人不认同
- 抖音/B 站有松弛感，但视频形态不适合作为操作参考

### 5.2 竞品深度分析

#### 竞品 A：小红书

**我们与它的根本区别**

小红书的核心商业逻辑是「美图 → 种草 → 消费」，它需要内容足够「好看」来维持平台的调性和广告溢价。这意味着：

- 出租屋里用普通锅具、在荧光灯下拍的西红柿炒鸡蛋，在小红书会因为「不够好看」而无人问津
- 步骤图不是平台鼓励的内容形态，因为对算法和广告不友好
- 创作者被隐性要求维持「精致人设」，与「松弛感」背道而驰

**我们的回应**：我们是「拍得不好看也没关系」的平台，步骤真实、过程真实比视觉精美更重要。这对普通年轻人有极强的吸引力。

#### 竞品 B：下厨房

**我们与它的根本区别**

下厨房的产品逻辑是数据库，内容是「食谱条目」，用户关系是工具性的而非社区性的。

- 没有场景感，所有菜谱平等，一道「出租屋红烧肉」和一道「米其林改良版红烧肉」没有情感上的区别
- 产品设计沉闷，没有任何「松弛感」或情绪价值的设计意图
- 核心用户是 30 ~ 45 岁的家庭主妇，年轻人进来感觉「这不是我的地方」

**我们的回应**：我们是社区，不是数据库。每篇内容背后有一个真实的人，有她/他具体的生活处境。场景标签是情绪入口，不只是功能筛选器。

#### 竞品 C：抖音/B 站 美食区

**我们与它的根本区别**

视频有场景感，但「不能边做饭边看视频」是硬伤。图文步骤才是厨房里真实有用的内容形态。

**我们的回应**：图文步骤 = 可以随时暂停翻查的「操作手册」。这是我们坚持图文优先而非视频的核心理由。

#### 竞品 D（国际参考）：Cookpad（日本）

Cookpad 证明了烹饪垂直社区的订阅制可行性，但它的产品调性接近下厨房，是「食谱工具」而非「生活社区」。中国本地化版本需要加入场景感和情感维度。

### 5.3 竞争定位图（修订版）

```
                    高情绪价值 / 场景感
                          │
           [我们的产品]    │    [小红书]
         步骤可复现 × 场景感│  精致 × 场景感
                          │  但无步骤结构
  工具导向 ───────────────┼──────────────── 娱乐/展示导向
                          │
           [下厨房]        │   [抖音][B站]
        工具 × 无场景感    │  场景感 × 不可复现
                          │
                    低情绪价值 / 工具理性
```

我们的坐标：**左上象限**，唯一做到「高情绪价值 + 步骤可复现」的产品。

## 6. 初步商业模式设想

### 6.1 场景化带来的新变现机会

与 v1.0 相比，场景分类体系打开了一条新的、更精准的变现路径：

**场景 × 品牌的精准匹配**

|场景|对应品牌类别|
|---|---|
|出租屋|小家电（铸铁锅、电饭锅、破壁机）、速食食材、外卖替代品|
|一个人的饭|精品食材、高端调味料、单人份厨具|
|露营野炊|户外炉具（Snow Peak、科帕奇）、户外餐具、速食便携食材|
|打包便当|便当盒品牌、保温容器、沙拉食材、健康零食|
|快手日常|半成品食材、调味料、速溶汤料包|
|减脂餐|健身代餐品牌、低卡调味料、鸡胸肉品牌、健康零食、体脂秤|
|家庭厨房|厨具品牌、厨房小家电、有机食材订阅|

这比「泛美食内容 + 泛广告」的模式精准 10 倍。一个在「露营野炊」场景下的品牌植入，其用户匹配度远高于任何综合平台的泛投放。

### 6.2 收入模式（维持 v1.0 框架，更新场景描述）

**路径 1：品牌场景化内容合作（首要 · P0）**

品牌与特定场景标签绑定，由平台帮助匹配对应场景的 T1 创作者创作原生内容。例如：某户外炉具品牌赞助「露营野炊」标签下的精选内容，在该场景 Feed 入口做原生曝光。

**路径 2：食材/器具电商导流（中期 · P1）**

每篇内容的食材列表和使用器具关联电商购买链接，按 CPS 收取佣金。场景化分类使「出租屋适用」「露营适用」的商品标签更清晰，转化率显著高于通用食谱平台。

**路径 3：创作者会员订阅（长期 · P2）**

为消费者提供付费会员，解锁高级场景筛选（如「出租屋 × 30 分钟 × 低卡 × 猪肉」组合筛选）、无限量收藏夹、专属创作者互动权益。

### 6.3 MVP 阶段变现优先级

```
上线即启动  →  品牌场景化合作（手动对接，1 ~ 3 个品牌试水）
上线 3 个月 →  电商 CPS 导流接入（与京东联盟 / 淘宝联盟对接）
上线 6 个月 →  会员订阅体系启动
```

## 7. 关键风险与假设

### 7.1 修订后的核心假设

1. **场景认同是比「食谱精准」更有力的拉新钩子**：年轻用户看到「出租屋类」标签的瞬间产生共鸣，比看到「番茄炒蛋」食谱更容易决定注册。
2. **松弛感降低了创作门槛**：不需要「够好看」才能发，普通人发的普通饭也值得被记录。这是拓宽 T3 创作者基数的关键。
3. **没有评论区反而是正向资产**：目标用户对社交压力敏感，无评论的纯点赞环境比「有评论但常被怼」的体验更好。
4. **图文步骤 > 短视频**（在「做饭时使用」这个场景下）：用户在真正烹饪时更需要静态步骤图，而不是需要暂停的视频。

### 7.2 核心风险

|风险|概率|缓解策略|
|---|---|---|
|「松弛感」定位被误解为「内容随意」，低质内容泛滥|中|结构化发布工具（至少 3 步、至少 1 张过程图）设定最低内容标准，同时保持门槛低|
|冷启动时场景标签下内容太少，体验差|高|上线前内部运营积累每个场景至少 30 篇种子内容|
|年轻用户只消费不发帖，UGC 不活跃|中|发帖激励（首发成就感设计）+ 降低发布步骤（最少 1 步也能发）|
|小红书对「出租屋」「露营」场景内容加强投入，填补空白|中|深耕步骤可复现性这个小红书无法做到的维度|

## 8. 立项决策建议

### 8.1 立项结论

**正式立项，进入 PRD 阶段。**

修订后的定位在以下维度优于 v1.0：

- 对 25 岁主力用户的情感共鸣更强，拉新转化效率更高
- 场景化分类打开了更精准的变现路径
- 与现有竞品的差异化护城河更宽，不是功能差异而是「气质差异」（更难被复制）

### 8.2 MVP 边界（与 v1.0 一致，场景标签新增）

**最终 In Scope**：场景标签（**8 个**，已确认）+ 食材/口味标签（多选）+ 双维度筛选

**永久 Out of Scope**：评论功能（不变）

### 8.3 产品气质一致性检查清单

每次产品决策前，对照以下六个词检查：

```
松弛  ✓ 这个设计让用户感到轻松吗？
真实  ✓ 这个设计鼓励真实内容而非表演吗？
有温度  ✓ 这个设计让人感到被看见、被理解吗？
可复现  ✓ 这个设计帮助用户真正做出那道菜吗？
有场景感  ✓ 这个设计强化了场景认同感吗？
不焦虑  ✓ 这个设计不会带来「不够好」的焦虑感吗？
```
