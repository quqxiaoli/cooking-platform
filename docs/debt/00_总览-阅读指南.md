## §1 四档判定规则（互斥，按顺序判）

| 档 | 名称 | 含义 | 默认处置 |
|---|------|------|---------|
| **1** | **设计失误** | 无正当理由违反 §3 评分卡列出的客观尺子（6 个模板 / 错误码段位 / Redis Key 命名 / Migration 命名 / Config-First / Lua 原子）。最该严判。 | 立即修 |
| **2** | **架构所限** | 受固定技术栈或三层架构约束，当前架构下没有更优写法，要改得动架构本身。 | 触发时修 / 永久接受 |
| **3** | **跨模块妥协** | 单看本模块这么写不合理，但为了和其它模块一致或解耦才让步。 | 触发时修 |
| **4** | **MVP 妥协** | 同架构同团队规模，给够时间就该改，且通常已在后续 step 计划里。 | Step N 重估 |

**判定优先级**（避免分档冲突）：

```
先看是不是档 1（违反自定规范）
  ↓ 否
再看是不是档 4（MVP 且计划内）
  ↓ 否
再看是不是档 3（为别的模块让步）
  ↓ 否
归档 2
```

## §2 条目固定字段（缺字段不合格）

每条债务条目必须用下列字段完整填写：

| 字段 | 说明 |
|------|------|
| **编号** | `TD-<模块缩写>-NN`，如 `TD-USER-01`、`TD-LIKE-03`、`TD-INFRA-01` |
| **档位** | 1 / 2 / 3 / 4 |
| **代码锚点** | 模块 + `文件:函数:行号`，可直接跳转；多锚点用列表 |
| **现状** | 现在怎么写（1–2 句） |
| **理想** | "好"应该怎么写（1–2 句） |
| **触发条件** | 这笔债何时真正咬人（如 `MAU>50w` / `接评论时` / `切 RabbitMQ 后` / `永不触发`） |
| **重构成本** | S（< 1 天）/ M（1–5 天）/ L（> 5 天，含设计） |
| **关联 ADR** | 回链 PRD v3.0 / 体检报告 / conventions 的章节编号；找不到写「无对应 ADR」 |
| **关联偏离点** | 回链已有偏离点编号（PRD §13 第 N 行 / progress/0X 偏离点表第 N 行）；无则写「无」 |
| **建议处置** | 立即修 / 触发时修 / 永久接受 / Step N 重估 |

**条目模板**（复制使用）：

```markdown
### TD-XXXX-NN · <一句话标题>

- **档位**：N
- **代码锚点**：`path/to/file.go:FuncName:LINE`
- **现状**：
- **理想**：
- **触发条件**：
- **重构成本**：S / M / L
- **关联 ADR**：PRD §X.Y 或「无对应 ADR」
- **关联偏离点**：PRD §13 第 N 行 / 「无」
- **建议处置**：
```

## §3 档 1 评分卡（设计失误的客观尺子）

凡违反下表任一条目即为档 1 候选；除非有显式 PRD 偏离登记授权，否则归档 1。

### 3.1 A 段：6 个模板锚点违规

| # | 模板 | 锚点文件 | 必含元素（违反即档 1） |
|---|------|---------|------------------|
| **A1** | Cache | `internal/cache/user_cache.go` | (a) 每个 Key 必有 `keyXxx(...)` 构造函数；(b) TTL 必须由调用方传入（来自 `cfg.Cache`），cache 包内禁止常量 TTL；(c) Not-found 返回 `ErrCacheNotFound` 哨兵，其他错误 `fmt.Errorf("redis CMD key: %w", err)`；(d) 共享 `redis.Client`，不 `Close()` |
| **A2** | Consumer | `internal/consumer/like_consumer.go` | (a) 构造签名 `New*Consumer(bus, dep, cfg config.*ConsumerConfig)`；(b) `Start(ctx)` → `eventCh = make(chan T, batchSize*4)` → `subWg`；(c) flushLoop 三路 select（eventCh / ticker / ctx.Done）；(d) ctx.Done 分支必须 `subWg.Wait() → consumer.DrainChan[T] → flush(context.Background())`；(e) 关闭日志固定格式 `zap.L().Info("xxx consumer drained", zap.Int("total_processed", n))`；(f) 禁止 labeled-for `drainLoop:` |
| **A3** | Repository | `internal/repository/user_repository.go` | (a) `type XxxRepository interface` 接口先行；(b) 小写 struct 实现 + `NewXxxRepository` 返回接口；(c) `ErrXxxNotFound` 哨兵，`gorm.ErrRecordNotFound` 必须翻译；(d) 全方法首参 `ctx`；(e) 上层永不直接拿到 `*gorm.DB` |
| **A4** | Service | `internal/service/user_service.go` | (a) `ctx` 透传到所有下游；(b) 业务错误返回 `*errcode.AppError`，基础设施错误 `fmt.Errorf` wrap；(c) 日志 `zap.L()` 字段化；(d) 不直接持有 `*redis.Client` / `*gorm.DB`；(e) 失败路径有补偿/回滚（如 SMS 失败删 code） |
| **A5** | Handler | `internal/handler/user.go` | (a) 严格 bind → validate → service → response 四步；(b) 错误统一 `response.FromError(c, err)`；(c) ctx 用 `c.Request.Context()`；(d) 不出现业务逻辑或基础设施调用 |
| **A6** | 外部依赖三件套 | `pkg/sms/{sender,mock,aliyun}.go` | (a) `sender.go` 定义接口；(b) `mock.go` dev 实现；(c) `aliyun.go` prod 实现；(d) provider 走配置开关；(e) AccessKey 走 env var，配置文件用 `<env:APP_*>` 占位 |

### 3.2 B 段：全局命名 / 段位 / 规范违规

| # | 规范 | 判定标准 |
|---|------|---------|
| **B1** | 错误码段位（PRD §7） | 必须符合 `XYYZZZ`；段位归属：通用 400000–409999 / 用户 410xxx / 内容 412xxx / 互动 430xxx / 关注 440xxx / 搜索 450xxx / 上传 460xxx / 审核 470xxx / 手机号加密 480xxx；越段或重号即档 1 |
| **B2** | Redis Key 命名（PRD §6.3） | `{module}:{purpose}:{key}` 模板；TTL 必须出现在 PRD §6.3 表，且通过 `cfg.Cache` 注入；命名漂移 / TTL 硬编码即档 1 |
| **B3** | Migration 命名 | `000NNN_<verb>_<resource>_table.up.sql` / `.down.sql`；编号递增、不跳号、不复用；违反即档 1 |
| **B4** | Config-First（conventions §1.1） | 任何 TTL / 批量 / 超时 / 阈值 / 外部地址必须进 `pkg/config/config.go`，禁止包级常量或魔数；违反即档 1 |
| **B5** | Lua 原子操作（conventions §1.3） | "读后写" / "条件写" 必须用 Lua（参照 `like_cache.go: addLikeScript / decrClampScript`）；用 pipeline 模拟即档 1 |
