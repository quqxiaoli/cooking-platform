**关键决策**：

1. **PresignedURL 替代 STS 临时凭证（核心偏离）**：PRD-Phase3 §8 写的是 STS。改用 PresignedURL 三点理由：①无 STS API 调用开销与延迟；②客户端纯 HTTP PUT，零 OSS SDK 集成；③object 级权限 + 15min TTL 已经足够安全（每个 URL 绑定固定 object_key，篡改 Content-Type 阿里云直接 403）。Step 12 PRD 收口阶段写入 v3.0。
2. **`pkg/oss` 三件套照搬 `pkg/sms` 模板**：`Client` interface + `NewClient(cfg)` factory + provider 字符串切换 mock/aliyun。dev 用 MockClient + 内嵌 listener，prod 用 AliyunClient + 真 SignURL，零业务代码差异。
3. **MockClient 内嵌 `net/http` listener（决策 2c 真实 PUT）**：监听 `cfg.OSS.MockListenAddr`（默认 127.0.0.1:18080），`PUT /<object_key>` 落到 in-memory map。让 verify_step9.sh 能跑完整 presign → 真实 PUT → callback 链路，而不依赖阿里云。listener 在 `NewMockClient` goroutine 启动、`Close` 时由 `http.Server.Close` 关闭（`closeOnce` 保证幂等）。
4. **`CallbackReq` 只含 nonce，server 是 PublicURL 的唯一来源**：不接受客户端回传 url。任何 client-supplied URL 都会被白名单怀疑；让 server 从 `NonceRecord` 读取自己签发时存的 URL，攻击面只剩"猜中别人 nonce"（2^128 entropy）。
5. **Nonce 原子消费用 Redis 6.2+ `GETDEL`**：替代旧版 GET-then-DEL 两步，关掉"两个 callback 同时来，各自 GET 到记录"的竞态。go-redis v9 直接 surface 为 `rdb.GetDel(...)`。
6. **`PublicURL` 在 presign 时就计算出来并写入 nonce 记录**：Mock 与 Aliyun 都用同一个 `buildObjectKey({purpose}/{user_id}/{yyyymm}/{uuid}.{ext})`，只拼一次，客户端可在 PUT 成功后乐观渲染，无需等 callback。
7. **白名单走"小写前缀匹配"**：`oss.IsAllowedURL` 把 url 与 prefix 都 `strings.ToLower` 后做 `HasPrefix`。`config.validate` 强制 prefix 配置为小写，保证 prod 审计时"配的就是匹配的"。空 url 视为"清空头像/封面"——合法。
8. **`PostRepository` 追加 `CreateWithSteps` 走事务，不改 `Create`**：`Create(ctx, p)` 留给 legacy 文字版 post（`Steps` 为空时走它）；有 steps 时走事务方法。事务里 post INSERT 拿到 LAST_INSERT_ID，再回填 `step.PostID`，批量 INSERT post_steps。半提交状态（post 已落 / steps 没落）永远观察不到。
9. **`StringArray` 自定义 sql.Scanner/Valuer 而非 `string`**：假设清单讨论时建议 MVP 简化用 `string` + service 层 marshal，小离要"哪个更好用用哪个"——更优雅的是 `[]string` 透明映射到 JSON 列。30 行边界代码换 service 全员零 `json.Marshal` 调用。空 slice 序列化为 `[]` 而非 `null`，callback 处不必 nil-guard。
10. **不引入 `event.upload` Topic**：Step 10 审核应在帖子发布时（`POST /posts`）发 PostEvent，AuditConsumer 拿 post + post_steps 一次性送审。为每张图发独立 upload 事件是过度设计。
11. **upload 模块装配 fail-fast**：`oss.NewClient` 失败直接 `log.Fatal`——server 起来但 upload 坏了，等于 Step 9 deliverable 没交付。日后若多 partner 部署可以放宽到 graceful degradation。
12. **`/presign` 走 PerUserKey 限流 30 次/小时**，`/callback` **不**限流：回调本就被单次 nonce + ownership check 卡死，加限流没价值；presign 才是真正的滥用入口（每次成功签发都占阿里云 STS quota 或污染 nonce 池）。
13. **新错误码 460xxx 独占整段**：延续 Step 7 起"每模块独占 4MM xxx 块"约定。401001 / 429001 / 410112 复用，不在 460xxx 重定义。
14. **shutdown 中 OSS Client 插在 HTTP→Consumer 之后、EventBus 之前**：确保不再有新 HTTP 上传请求时才关 listener；EventBus 留到最后是因为 Consumer 关闭前仍要 publish ack。
15. **`buildObjectKey` 放在 `pkg/oss/mock.go` 而非 `client.go`**：它是 mock 与 aliyun 共用的拼 key 函数，本可单独成 file，但项目阶段无第三个 provider，放在 mock.go 与 `extFromContentType` 邻居关系更合理（都属于"derive key from params"语义簇）。Step 13 接入第二个真 provider 时再单独拆 file。

**本步与 PRD 的偏离点**：

| PRD 章节 | 实现差异 | 选择此方案的原因 |
|---|---|---|
| PRD-Phase3 §8 OSS 直传架构图 STS 临时凭证 | 实装 PresignedURL（Aliyun OSS SDK `Bucket.SignURL` 签发 PUT URL，TTL=15min，object 级权限） | ①无 per-presign STS API 调用开销；②客户端纯 HTTP PUT，零 OSS SDK 集成；③object 级权限 + 短 TTL 在 MVP 流量下足够安全；STS 留作 Step 13 生产升级时的可选项 |
| PRD-Phase2 §4 F-C01 "步骤列表 1..30 步，每步图 0..3 张" | 实装为 `post_steps` 子表 + 每步 `image_urls JSON` 列；上限 30 步、每步 ≤3 图、文字 ≤500 字 | PRD §4 注释明确指引 Step 9 落地子表；JSON 列承载 ≤3 张 URL 数组比另开 `post_step_images` 二级子表更轻；`uk_post_step(post_id, step_no)` 提供并发兜底 |
| PRD-Phase3 §6.3 Redis Key 命名全表 | 新增 `upload:nonce:{nonce}` Key（VALUE = JSON{user_id, object_key, public_url, purpose, content_type, size}，TTL=15min） | Presign 与 Callback 解耦必须有原子状态承载；GETDEL 原子消费杜绝重放与竞态；TTL 与 PresignTTL 共享配置项 `oss.presign_ttl` |
| PRD-Phase2 §3 F-U04 "头像上传成功后 5 秒内全站更新（CDN 缓存刷新）" | UpdateProfile 同步落库 + 主动 `userCache.DeleteUserInfo` 失效本地缓存。CDN 刷新走 OSS 默认（无需服务端发起） | 服务端只能控自家缓存；CDN 刷新由 OSS 端按对象覆盖语义触发，且 MVP 用 mock 模式无 CDN，prod 接入阿里云 OSS 自带 |
| PRD-Phase2 §4 F-C01 "每步图片上传后顺序绑定" | DTO `Steps` 入参顺序即步骤顺序（数组下标 → `step_no = i+1`） | 比客户端显式传 `step_no` 更简洁；前端通过 reorder 数组即可改步骤顺序 |
