## 已完成的步骤详情

### 第 1 步：项目初始化（2026-04-23）

**产出文件**：
- `go.mod`
- `Makefile`
- `docker-compose.yml`（开发版：单 MySQL + 单 Redis）
- `.env.example`、`.gitignore`
- `configs/config.yaml`、`configs/config.prod.yaml`
- `deploy/mysql/dev.cnf`
- `pkg/config/config.go`
- `pkg/logger/logger.go`
- `pkg/errcode/errcode.go`
- `pkg/response/response.go`
- `internal/middleware/request_id.go`、`logger.go`、`recovery.go`、`cors.go`
- `internal/handler/health.go`
- `cmd/server/main.go`
- 各业务包占位 `doc.go`（handler、service、repository、model、cache、event、consumer、pkg 各子包）
- `scripts/migrate.sh`、`scripts/seed.sh`、`scripts/backup.sh`

**关键实现决策**：
- 使用 `gin.New()` 而非 `gin.Default()`，自行注册 zap-based 中间件，保证日志结构化且携带 request_id
- `go.mod` 中 lumberjack 包名为 `gopkg.in/natefinch/lumberjack.v2`（natefinch，不是 natefinsh）
- docker-compose.yml 开发版只有单 MySQL + 单 Redis，无主从、无 Sentinel、无 RabbitMQ
- MySQL 容器挂载 `deploy/mysql/dev.cnf`，开启 `ngram_token_size=2` 和 GTID，与生产配置保持一致
- Redis 开发环境不设密码
- `initRedis` 返回 `*redis.Client`，第 14 步切换 Sentinel 只改该函数体,业务代码零修改

**已验证**：

```bash
make docker-up   # MySQL + Redis 健康启动
make run         # 服务监听 :8080，mysql connected、redis connected
curl localhost:8080/health
# {"code":0,"msg":"ok","data":{"status":"ok"},"request_id":"..."}
curl localhost:8080/readiness
# {"code":0,"msg":"ok","data":{"mysql":"ok","redis":"ok"},"request_id":"..."}
```

**遗留问题**：
- `docker-compose.yml` 顶部 `version` 字段会触发 Docker Compose 警告（obsolete），不影响运行，下次顺手删掉即可
- 本地 Homebrew 安装的 MySQL 因 3306 端口冲突已 `brew services stop`，当前数据库完全由 Docker 容器提供
