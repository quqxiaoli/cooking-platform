## 三、域名 + HTTPS（2026-05-20，阶段 1，随 Step 20 一起落地）

### 域名变更

- 原计划：`laidbackck.cn`，但 .cn 注册局审核耗时不可控
- 实际：改为 **`mellowck.com`**（阿里云万网购买）
- 理由：laidback.com 被占；mellow 气质更贴《摇曳露营》原型；**.com + 香港服务器 → 不触发 ICP 备案**，省去原计划 20 天工信部审核卡点

### DNS 解析

| 项 | 结果 |
| --- | --- |
| A 记录 | `@` + `www` 两条 → `47.238.29.251`，生效 < 10 分钟 |

### Let's Encrypt 证书

| 项 | 结果 |
| --- | --- |
| 证书 | SAN 证书覆盖 `mellowck.com` + `www.mellowck.com`，certbot **webroot 模式**（不停 nginx），90 天有效期 |
| certbot 部署 | **host apt 安装**（不用 Docker certbot 镜像）；证书直接落 `/etc/letsencrypt/`，nginx 容器只读挂载 |
| 自动续期 | 可挂 cron `certbot renew --quiet && docker exec cooking-nginx-prod nginx -s reload`（推荐每周一次） |

### nginx 改造

| 项 | 结果 |
| --- | --- |
| 80 端口 | 拆分：只保留 `/.well-known/acme-challenge/` + 其他全部 301 跳 HTTPS |
| 443 端口 | 承载所有业务 location（/metrics 屏蔽 / /health / /api/v1/auth/send-code / /api/ / /） |
| SSL 配置 | **Mozilla Intermediate**：TLSv1.2 + TLSv1.3、ECDHE-only 密码套件（免 dhparam）、`ssl_session_cache shared:MozSSL:10m` |
| HSTS | `max-age=31536000`（1 年），跑顺畅后可升级 preload 名单 |
| 卷挂载 | nginx 服务追加 `/etc/letsencrypt:/etc/letsencrypt:ro` + `./deploy/certbot/webroot:/var/www/certbot:ro` |

### 验证

- **SSL Labs 评级：A+**
- Mac 本地浏览器 `https://mellowck.com` / `https://www.mellowck.com` 均正常加载

## 四、关键技术决策（HTTPS 段）

| 决策 | 选择 | 理由 |
| --- | --- | --- |
| certbot 部署位置 | host apt 安装（不用 Docker certbot 镜像） | 续期 cron 在 host 上更稳定；证书直接落 `/etc/letsencrypt/`，nginx 容器只读挂载 |
| HTTP→HTTPS 策略 | 保留 80 端口但只服务 ACME，其他全 301 跳转 | 续期需要 80 可达；业务流量强制 HTTPS |
| SSL 配置标准 | Mozilla Intermediate | 兼容到 TLSv1.2，覆盖 22-28 岁目标用户全部设备；Modern 会拒掉旧 Android |

## 本步与 PRD 的偏离点

| PRD 章节 / 规则 | 实现差异 | 选择此方案的原因 |
| --- | --- | --- |
| Step 18 域名 `laidbackck.cn` | 改为 **`mellowck.com`**（阿里云万网购买） | laidback.com 被占；mellow 气质更贴《摇曳露营》原型；.com + 香港服务器免备案，省 20 天工信部审核卡点 |
| Step 18 HTTPS 方案 | certbot webroot 模式 + Mozilla Intermediate + HSTS max-age 1 年 | webroot 续期不需要停 nginx；Intermediate 兼容到 TLSv1.2（覆盖目标用户全设备）；HSTS 1 年是 SSL Labs A+ 入门门槛 |
| nginx.conf 部署结构 | 拆分为 80 端口（ACME + 301）+ 443 端口（业务全部）两个 server 块 | 80 端口必须可达（certbot 续期）；业务流量强制 HTTPS；server_name 从通配 `_` 改为 `mellowck.com www.mellowck.com` |
| nginx.conf 卷挂载形式 | 仍是单文件 `bind mount`（未改目录挂载） | 改挂目录涉及全员 nginx 配置组织重构，工作量大；本步保留单文件 + 已知 `force-recreate` 应急路径；改造挂为遗留债务（见 Step 20 遗留债务 #3） |
