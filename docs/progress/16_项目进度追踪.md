**关键决策**：

1. **pkg/metrics 包 + Init(namespace) 模式**：所有指标定义为包级 nil 指针，Init() 一次性注册到默认 registry。避免 init() 隐式注册（测试友好），同时 namespace 从 cfg.Metrics.Namespace 注入（Config-First 规则）。未调用 Init() 时指针为 nil，各埋点均有 nil-guard，不 panic。

2. **MySQL pool 用 pull-based Collector**：在 Prometheus scrape 时调用 sql.DBStats()，无需后台 goroutine 或定时器。拉模型天然与 Prometheus 架构对齐，数据永远是抓取时刻的最新值。

3. **纳入 P1（Redis hook + MySQL pool）**：代码量约 120 行，但面试中可以讲"如何对第三方 client 做非侵入式插桩"。Redis Hook 展示了装饰器模式；MySQL Collector 展示了 pull-based 采集与 push-based 采集的区别。

4. **/metrics 不通过 Nginx 代理**：Prometheus 直接访问 app1:8080/metrics 和 app2:8080/metrics（Docker 内网），/metrics 不对外公开。per-instance 指标（goroutine count、GC pause）也因此可分别查询。

5. **Grafana 自动 provisioning**：datasource 和 dashboard 通过 deploy/grafana/provisioning/ 目录在容器启动时自动注入，无需手动操作 Grafana UI，符合 GitOps 理念。
