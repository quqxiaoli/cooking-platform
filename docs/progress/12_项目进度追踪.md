- `Makefile`（修改：追加 `verify-step12` 目标）

**关键决策**：

1. **470xxx/480xxx 前缀不改**：这两段是 server-internal 错误码，HTTP 500。用户确认保持 4 前缀（与其他所有模块一致），通过包注释明确说明 X 前缀约定，不做运行时破坏性变更。
2. **sanitizeQuery 防御性设计**：当前所有 phone 路由均为 JSON body，query string 里不会出现 phone。但中间件层提前 block，防止未来路由误设计导致泄露。
3. **MaskToken 策略：≤8 位全遮**：token 前 8 位的信息量（用于日志关联）对短 token 没有意义，全遮是更安全的选择。长 token（JWT 通常 >100 字符）保留前 8 位足够做跨日志关联。
4. **Security 中间件无配置开关**：所有四个响应头对 dev 和 prod 都安全，不需要 console/prod 分支，避免 dev 环境掩盖 prod 安全问题。
