**关键决策**：
1. **JWT 算法 HS256 + alg confusion 防御**：parse 时硬校验 `*jwt.SigningMethodHMAC`，挡掉 alg:none 攻击
2. **Claims 结构 uid (int64) + jti (UUIDv4)**：uid 替代标准 sub 字段，省 strconv 转换；jti 客户端生成，零网络往返
3. **JWT 黑名单 TTL 动态计算**：登出时取 access token 剩余 ttl，过期后自然清理 Redis
4. **手机号 + 验证码登录"找不到自动注册"**：去除独立注册接口，UX 简洁
5. **Refresh 不旋转旧 token**：MVP 不做 refresh token rotation，第 11 步加密迁移时再评估
6. **Auth 中间件只注入 user_id 不预加载 User**：节省每请求一次 DB 查询，热路径友好
7. **限流"fail open"**：Redis 故障时放行所有请求，避免单点故障让 API 全挂
8. **SMS 限流三维**：phone 短窗 60s + phone 日限 5 次 + IP 日限 10 次，固定窗口实现（非滑动窗口）
9. **IP 哈希后做 Key**：避免明文 IP 进 Redis（GDPR-aligned 习惯）
10. **滑动窗口 ZAdd 仅在 allowed 时执行**：拒绝的请求不消耗窗口槽位，防"窗口卡满"攻击
11. **验证码用 crypto/rand 而非 math/rand**：math/rand 可被预测，攻击者可枚举下一码
12. **手机号 phone_encrypted 第 3 步存明文**：第 11 步迁移到 AES-GCM，phone_hash 字段保持不变
13. **migrations 用 SQL 而非 GORM AutoMigrate**：可审计、可回滚、可 review
14. **Repository UpdateProfile 用 map 而非 struct**：避免 GORM struct 更新跳过零值的陷阱
15. **config Load() 加 validate**：JWT secret 弱、TTL 配错等致命错误启动时立即暴露

**关键决策**：

1. **Post.Content 直接落 posts 表 TEXT 列**：避免 MVP 阶段为了"工整"先建 post_steps 子表带来的复杂度；第 9 步引入图片上传后，若结构化步骤需要再做一次迁移
2. **错误码段位 412xxx**：用户 410xxx，411 留给用户扩展（账号恢复、设备管理），412 给内容；保留命名空间，未来扩展不挤位
3. **游标分页 = `created_at` UnixMilli 字符串**：稳定、可比较、无 offset 深翻页性能塌陷；`""` 表示第一页，编码不做 base64 便于调试
4. **is_visible 冗余字段**：MVP 直接置 1，第 10 步审核接入后由 AuditConsumer 维护；Feed 查询固定 `WHERE is_visible=1`，索引命中率 100%
5. **Feed 缓存版本号机制**：`feed:ver` 全局 INCR，旧 key 通过 300s TTL 自然过期；零 SCAN、零阻塞，胜过"DEL pattern"做法
6. **PVEvent 第 4 步只发不消费**：路径已经全部接入，第 5 步插入 PVConsumer 即可；MVP 期 view_count 始终为 0，已登记偏离点
7. **作者快照 N+1 查询**：每篇 post 一次 userRepo.FindByID；MVP size≤50 可接受，FindByIDs 批量加载留到关注 Feed 出现时
8. **作者主页 URL `/users/:id/posts`**：与 PRD-Phase3 §7.1 follow 路由风格一致，比 `/feed?author_id=` 更语义化
9. **删除作者后内容仍可见**：load 不到作者时返回"（用户已注销）"占位，content 不消失
10. **PV 去重双桶**：登录用户 `pv:dup:{post_id}:u{user_id}`，游客 `pv:dup:{post_id}:i{ip_hash}`；命名空间隔离防止 UID 与 IP hash 碰撞
11. **EventPublisher 注入而非整个 EventBus**：service 只发不订阅，依赖最小化（接口隔离原则）
12. **service 显式写 `time.Now()` 给 CreatedAt/UpdatedAt**：GORM 见到 struct tag 含 `default:CURRENT_TIMESTAMP(3)` 会跳过 autoCreateTime 填充也不回读，否则返回的 created_at 是零值（-62135596800000）
13. **滑动窗口限流挂在 Auth 之后**：未登录请求先被 401 挡掉，省一次 Redis 查询
14. **复合索引顺序"等值在前、范围在后"**：`idx_scene_visible_created` 三列经典最左前缀，scene 等值匹配 → is_visible 等值 → created_at 范围
15. **FULLTEXT 索引第 4 步提前建好**：搜索第 7 步才用，但百万行后 ALTER TABLE 加 FULLTEXT 会锁表过久

## 第 4 步踩坑记录

1. **GORM autoCreateTime 与 default tag 冲突**：当 `CreatedAt` 字段同时含 `default:CURRENT_TIMESTAMP(3)` tag 时 GORM 跳过 Go 层填充也不回读 DB 值，service 拿到的是 time.Time 零值（UnixMilli = -62135596800000）。修复：service 创建 Post 时显式 `time.Now()`。
2. **Redis 端口冲突（brew vs docker）**：本机 Homebrew Redis 与 Docker Redis 同时监听 6379（brew 绑 127.0.0.1，docker 绑 0.0.0.0），内核优先具体接口路由 → server 全程连到 brew Redis，docker 这边一直空。修复：`brew services stop redis` + 重启 server。已写入"开发环境约定"防御措施段落。
3. **复制粘贴中文引号污染 token**：手动 `TOKEN="..."` 时贴入末尾的中文 `"` 让 zsh 进入 dquote 等待，TOKEN 实际值含中文引号 + 换行符 → JWT.Parse 返回 401003。修复：用 `TOKEN=$(curl ... | jq -r ...)` 一行命令避免手动复制。
